Lab - Native Android Development

Mission: In this lab, you will build and run a simple 'Hello World' app using Android native tools. Your goal is to become familiar with the Android toolset and the steps that native developers must endure to create even simple Android projects.

1. At a command prompt, create a folder in your home directory named HelloWorldAndroid
You will store the files for your Android project in that folder.
2. At a command prompt, enter 

android list targets
To get a list of Android API targets available on your system. You'll need to use a valid target number in the next step.
3. Enter: 

android create project --target TARGET_NUMBER
--name MyAndroidApp 
--path ./HelloWorldAndroid
--activity MyAppActivity 
--package com.app.myandroid
To create a new project, where TARGET_NUMBER is a valid target number as determined in step 1.
4. Switch into the HelloWorldAndroid directory and explore the files that were created.
The various files and folders contain the
5. With a text editor, open HelloWorldAndroid/src/com/app/myandroid/MyAppActivity.java
This Java file is used to open your app's "activity" which in Android represents a component of an application with which the user can interact. As a native developer, you would modify this file with the Java code that enables your app's functionality.
6. Close that file. Then, with your text editor, open HelloWorldAndroid/res/layout/main.xml. Modify the "Hello World" text to a message of your choice. Save and close the file.
This is the XML layout file for your app. As a native developer, you would modify this file and files like it to create your app's user interface.
7. In the HelloWorldAndroid directory, enter 

ant debug
The ant build script will build and sign the resulting apk file. You will see the message “BUILD SUCCESSFUL” upon completion. If you get an error regarding an expired Debug Certificate, remove the file debug.keystore from the ./android directory. A new version will be created when you next execute “ant debug.”
8. Enter 

android
To open the Android SDK Manager application. You will use this app to create an Android Virtual Device (an emulator) in which to run your app.
9. From the Tools menu, choose AVD Manager to open the window shown here:
10. Click New and create an AVD with a target API level equal to or higher than your app's target. Examine the other options, such as the SD card size, skin (screen size and resolution), and emulated hardware features.
11. Click Start, then click Launch.
To start your AVD. It will take a few minutes, depending on the speed of your computer, to fully start up.
12. Back at the command prompt, in the HelloWorldAndroid folder, enter 

adb install bin/MyAndroidApp-debug.apk
To download and install your app on the emulator. If you need to reinstall it later, you can add the -r switch to the command above, or remove the app with adb uninstall com.app.myandroid (notice that you use the package name to uninstall, but the file name to install).
13. In the emulator, click the Tray button, then click MyAppActivity
To launch the app. Your custom text should be displayed on the screen.
14. Back at the command prompt, enter 

ddms
To open the Dalvik Debug Monitor. The Dalvik Debug Monitor is your window to the AVD. Here you can inspect your application, view logs, look at threads, and depending on your hardware setup, simulate things like GPS coordinates or making calls. You should be able to see your application in the process list as com.app.myandroid. Click it and explore the threads and other debug information. You can also stop the process from here if needed.
15. Close DDMS, the emulator, the AVD manager, and all open windows.

Summary

This lab was a brief and simple introduction to the Xcode development tools.
