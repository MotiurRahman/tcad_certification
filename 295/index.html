Android Deployment

Let's talk first about deploying your app for testing on Android devices

Android deployment for testing

Simple, two-step process for Android

Configure device for testing

First, configure device for testing
Windows users: need to install drivers specific to your device
Linux users: need to create USB config
See the wiki

Local Android deployment

  1. Connect device to USB port
  2. From Run menu, choose Android Device
Android deployment for testing is pretty simple
Can also manually install the APK via ADB
Or by transferring apk to device and opening with a file manager
Ti Studio signs APK with a debug compatible certificate. You could go through the signing steps but that's unnecessary

Remote Android deployment

Get the APK to the user's device


Email


Post on a web site


Distribution framework


Just get the APK onto the user's device
Use the apk-debug version to skip code signing steps
Email, post on a web site, share through Dropbox
Over-the-air system like HockeyKit can automate deployment and updates

iOS Deployment

iOS deployment for testing

Six step of Apple's process — don't go in-depth here

Apple Developer Program

Free to register, download SDK


Annual fee for iOS Developer program


Individual, Company, Enterprise, & University programs


You can register for free
Need to pay to publish and distribute apps
Various levels with varying documentation requirements and fees

Developer's certificate

Certificate identifies you as the developer
Create with Keychain Access
Upload CSR file to provisioning portal
Download and install the resulting certificate into login keychain
Also need Apple's WWDR certificate

Register devices

Register devices used for testing
Need the UDID
Get that from Xcode or iTunes
Enter at Provisioning Portal

Create an AppID

Unique identifier for your app


Bundle Seed ID - 10 character universally unique


CF Bundle Identifier - app name or wildcard


Bundle Identifier must be unique for Push, In-App purchases, Game Center


Bundle Seed ID generated by the Provisioning Portal
CF Bundle Identifier you specify
Could name a specific app
Could be asterisk (wildcard)
Need specific name, not wildcard for Push Notification, In-App purchases, and Game center

Provisioning profile

Provisioning profile = collection of your certificate, app ID, and device IDs
Install to device to enable testing
You'll create a similar provisioning profile for final distribution

Install the provisioning profile



Option 1: Drag & Drop to Xcode


Ultimately, you need to install to Xcode
You can drag & drop
or install via Ti Studio (next slide)

Install the provisioning profile



Option 2: Install via Titanium Studio


If not installed via Xcode, click Upload and Ti Studio will install for you
Ti Studio should recognize certificates in your keychain

Installing to local test device

Google Play

(formerly the Android Market)

Android deployment for release

Steps of Android release process — don't go into depth here

Google Play requirements

Google account


Android Developer account


One-time fee


SDK tools: keytool, jarsigner, zipalign


Need a Google account (Gmail, Google Docs)
Sign up at market.android.com/publish
One-time $25 fee
Need an Google Checkout Account if you'll be selling your app for a fee
Studio provides graphical interface to command-line tools. But you still might need to use the SDK tools, such as to verify APK file

Generate key pair and certificate

When you choose to Distribute to the Android Market, Studio provides this dialog for creating a certificate pair and keystore.
Used to have to do this with the keytool command
Alias varies and is specific to your app
Can store multiple certificates in the same keystore
Make backup copies of your keystore file and save in safe location

Verify keystore

Optional step, but you should do it anyway
If you lose the keystore or it's invalid, you won't be able to issue updates in the market
Point out keytool command
Want to see 'Valid From' and date 25 years from now or longer

Sign your application

From Deploy menu, choose Distribute — Android
You'll specify your keystore location, password, and alias

Verify signed app

Again, this is optional but recommended
Use jarsigner to verify apk file
Looking for 'jar verified'
Might need to use zipalign tool. Android.com site will inform you if you do.
Shouldn't be needed after Titanium 1.4

Uploading and publishing

Upload at market.android.com/pubish
Some required info
Rest optional
Free or set price if you have Google Checkout account

iTunes AppStore

iOS deployment for release

Similar steps for release as testing
Done two steps already

Distribution certificate

App needs to be signed with distribution certificate to be submitted to App Store
Corporate / team environment:
- each developer creates their own CSR/Developer certificate
- team manager creates distinct CSR/Distribution certificate
- developer certs can be revoked without affecting other developers or distribution

Individual developers
- can re-use development CSR to create distribution certificate
- could create new if you'd like
Must be logged in as Agent role (one per developer account) to create
Install to keychain on machine that will be used for final build

Distribution Provisioning Profile

Not tied to specific devices
Required for App Store submission
Download the resulting mobileprovision file

Build for Distribution

Install into Ti Developer
- Upload button installs to Xcode or do separately
Should recognize certificates in your keychain
Package button builds your app, embeds the certificates, and creates IPA that you'll upload to the App Store

iTunes Connect

Portal for application management


Sales statistics


Contracts, taxes, payments, banking connections


Need to create application within iTunes Connect


Creating an app in iTunes Connect

Bundle ID and suffix must match what you enter in Ti Studio
Step through series of pages to define price, content rating, etc.
You'll need a 512x512 icon and at least one screenshot

iTunes Graphics Required

  • At least one screenshot per device type your app supports
  • A 1024 x 1024 app icon
  • App icons, launch images, supporting graphics appropriate for the devices your app supports

See http://appcel.us/e9RqM for complete list

For example, for iPhone, you'll need iPhone4 & iPhone5 sized screens, the large icon, plus launch images, iTunesArtwork file, and appicons in various sizes.

Validating and uploading

Xcode organizer, Archives window
Validate app — checks for app in iTunes Connect, required graphic assets, signing, etc.
Click Validate to validate the app and its configuration on the AppStore
Click Submit - to submit. You can also upload using the Application Loader (in Developer/Applications/Utilities folder)

App Store Review Requirements

Many requirements - see iOS Dev Center for full details


Must be useful, well-designed, run without errors


No downloading code


You must own trademarks


No hidden features, use of non-public APIs

Feel free to describe relevant experiences

Limited to 100 devices per year
